# 02. 자료구조 활용 — 언제 무엇을 꺼내 쓸 것인가

> **이 문서에서 배우는 것**
> - 자바 컬렉션의 **선택 기준**: List·Map·Set·Stack·Queue·Deque·PriorityQueue
> - 각 자료구조의 **핵심 연산 복잡도** (이게 곧 시간초과 여부를 가른다)
> - 문제의 신호를 보고 **"이건 스택/힙/해시"** 를 판단하는 감각

알고리즘의 절반은 "적절한 자료구조를 고르는 것"입니다.
같은 문제도 배열로 풀면 O(N^2), 해시로 풀면 O(N)이 됩니다. 도구를 아는 사람이 이깁니다.

> 💡 이 문서는 [01번(시간복잡도)](./01_기초_시간복잡도와_입출력.md)을 먼저 읽었다고 가정합니다.
> "O(1) 조회", "O(N) 탐색" 같은 말이 편해야 합니다.

---

## 0. 한눈에 보는 선택 지도

| 하고 싶은 일 | 자료구조 | 핵심 연산 복잡도 |
|---|---|---|
| 순서대로 담고 인덱스로 접근 | `ArrayList` | 조회 O(1), 끝에 추가 O(1) |
| 빠른 존재 확인 / 중복 제거 | `HashSet` | 추가·조회 O(1) |
| 키→값 매핑 / 빈도 세기 | `HashMap` | 추가·조회 O(1) |
| 마지막에 넣은 걸 먼저 꺼냄(LIFO) | `Deque` (스택 용도) | push/pop O(1) |
| 먼저 넣은 걸 먼저 꺼냄(FIFO) | `Deque` / `ArrayDeque` (큐 용도) | offer/poll O(1) |
| 가장 작은/큰 값을 계속 꺼냄 | `PriorityQueue` (힙) | 추가·삭제 O(log N) |
| 키를 정렬된 상태로 유지 | `TreeMap`/`TreeSet` | 추가·조회 O(log N) |

> ⚠️ **자바 관례 주의**: 스택과 큐 모두 `ArrayDeque`로 쓰는 것을 권장합니다.
> - 옛날 `Stack` 클래스는 동기화 때문에 느리고 구식입니다. → `Deque<Integer> stack = new ArrayDeque<>();`
> - `LinkedList`도 큐로 되지만 `ArrayDeque`가 더 빠릅니다.

---

## 1. List — 순서 있는 목록

가장 기본. 크기가 자동으로 늘어나는 배열이라고 생각하면 됩니다.

```java
List<Integer> list = new ArrayList<>();
list.add(10);          // 끝에 추가 O(1)
list.add(20);
int x = list.get(0);   // 인덱스 조회 O(1) → 10
list.set(1, 99);       // 값 변경
list.size();           // 개수

for (int v : list) System.out.println(v);   // 순회
```

> 📌 **함정**: `list.contains(x)`(포함 여부)와 `list.remove(Integer)`(값 삭제)는 **O(N)** 입니다.
> "특정 값이 있는지 자주 확인"해야 하면 List가 아니라 **Set/Map**을 쓰세요.
>
> 또 하나: `list.remove(2)`는 **인덱스 2 삭제**, `list.remove(Integer.valueOf(2))`는 **값 2 삭제**입니다.
> int를 remove하면 인덱스로 해석되니 주의!

고정 크기로 충분하고 성능이 중요하면 그냥 **원시 배열 `int[]`** 가 가장 빠릅니다.

---

## 2. Map — 키로 값을 찾기 / 빈도 세기

코테에서 **가장 자주 쓰는 무기**입니다. "빈도 세기", "본 적 있나?", "짝 찾기"의 핵심.

```java
Map<String, Integer> count = new HashMap<>();

// 빈도 세기의 정석 — getOrDefault
String[] words = {"apple", "banana", "apple", "apple"};
for (String w : words) {
    count.put(w, count.getOrDefault(w, 0) + 1);
}
System.out.println(count.get("apple"));   // 3

// 더 짧게 (Java 8+)
Map<String, Integer> c2 = new HashMap<>();
for (String w : words) c2.merge(w, 1, Integer::sum);

// 순회
for (Map.Entry<String, Integer> e : count.entrySet()) {
    System.out.println(e.getKey() + " → " + e.getValue());
}
```

**대표 활용 — 두 수의 합(Two Sum):** "합이 target인 두 수가 있는가?"를 O(N^2) 이중 반복 대신 O(N)으로.

```java
// 지금까지 본 수를 Map에 저장하며, target - 현재값 을 이미 봤는지 O(1)로 확인
Map<Integer, Integer> seen = new HashMap<>();   // 값 → 인덱스
for (int i = 0; i < nums.length; i++) {
    int need = target - nums[i];
    if (seen.containsKey(need)) { /* 찾음: seen.get(need), i */ break; }
    seen.put(nums[i], i);
}
```

이 "이미 본 것을 해시에 담아 O(1)로 되묻기" 패턴은 코테 단골입니다. 꼭 익히세요.

---

## 3. Set — 중복 제거 / 존재 확인

값의 **유무만** 중요하고 개수는 상관없을 때. `HashSet`은 추가·조회가 O(1).

```java
Set<Integer> set = new HashSet<>();
set.add(3);
set.add(3);              // 중복은 무시됨
System.out.println(set.size());        // 1
System.out.println(set.contains(3));   // true, O(1)

// List의 중복 제거
List<Integer> unique = new ArrayList<>(new HashSet<>(listWithDup));
```

> 💡 정렬된 순서가 필요하면 `TreeSet`(자동 정렬, O(log N)), 삽입 순서 유지가 필요하면 `LinkedHashSet`.

---

## 4. Stack — 마지막에 넣은 것을 먼저 (LIFO)

"가장 최근 것", "짝 맞추기", "되돌리기(undo)", "뒤에서부터 처리"의 신호가 보이면 스택.

```java
Deque<Integer> stack = new ArrayDeque<>();   // Stack 클래스 대신 이걸 권장
stack.push(1);
stack.push(2);
System.out.println(stack.peek());   // 2 (맨 위 확인, 꺼내진 않음)
System.out.println(stack.pop());    // 2 (꺼냄)
System.out.println(stack.isEmpty());// false
```

**대표 활용 — 올바른 괄호 검사:** 여는 괄호는 push, 닫는 괄호를 만나면 pop해서 짝을 맞춥니다.

```java
static boolean isValid(String s) {
    Deque<Character> st = new ArrayDeque<>();
    for (char c : s.toCharArray()) {
        if (c == '(') st.push(c);
        else {                       // c == ')'
            if (st.isEmpty()) return false;   // 닫는데 열린 게 없음
            st.pop();
        }
    }
    return st.isEmpty();             // 안 닫힌 게 남았으면 실패
}
```

## 5. Queue — 먼저 넣은 것을 먼저 (FIFO)

"순서대로 처리", "대기열", 그리고 **BFS(너비 우선 탐색)** 의 엔진. ([05번 문서](./05_그래프탐색_DFS_BFS.md)에서 핵심으로 등장)

```java
Queue<Integer> queue = new ArrayDeque<>();
queue.offer(1);      // 뒤에 추가 (= add)
queue.offer(2);
System.out.println(queue.peek());   // 1 (맨 앞 확인)
System.out.println(queue.poll());   // 1 (맨 앞 꺼냄)
```

> 📌 `Deque`(양쪽 끝 삽입/삭제 가능)는 스택으로도 큐로도 쓸 수 있는 만능입니다.
> `ArrayDeque` 하나로 둘 다 해결하는 걸 권장합니다.

---

## 6. PriorityQueue — 항상 가장 작은/큰 것부터 (힙)

넣은 순서와 무관하게, 꺼낼 때 **항상 우선순위가 가장 높은 원소**가 나옵니다. 내부는 힙(heap), 추가·삭제 O(log N).
"가장 작은 값을 계속 꺼내기", "K번째로 큰 값", "다익스트라 최단경로"의 핵심.

```java
// 기본: 최소 힙 (가장 작은 값이 먼저)
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(5); minHeap.offer(1); minHeap.offer(3);
System.out.println(minHeap.poll());   // 1 (가장 작음)
System.out.println(minHeap.poll());   // 3

// 최대 힙 (가장 큰 값이 먼저) — 비교자를 뒤집음
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Collections.reverseOrder());
maxHeap.offer(5); maxHeap.offer(1); maxHeap.offer(3);
System.out.println(maxHeap.poll());   // 5
```

**대표 활용 — K번째로 큰 값:** 크기 K짜리 최소 힙을 유지하면, 힙의 top이 곧 K번째로 큰 값.

```java
PriorityQueue<Integer> pq = new PriorityQueue<>();   // 최소 힙
for (int x : nums) {
    pq.offer(x);
    if (pq.size() > k) pq.poll();   // K개만 유지 (작은 것부터 버림)
}
// pq.peek() == K번째로 큰 값.  전체 정렬 O(N log N)보다 유리할 수 있음(O(N log K))
```

> ⚠️ `PriorityQueue`는 **전체가 정렬되어 있지 않습니다.** `poll()`로 꺼낼 때만 최솟값이 보장됩니다.
> `for (int x : pq)` 로 순회하면 정렬 순서가 아니니 주의!

전체 실행 예제는 [샘플/src/Ex05_PriorityQueue.java](./샘플/src/Ex05_PriorityQueue.java)에 있습니다.

## 7. 복잡도 비교 요약표 (외워두면 판단이 빨라짐)

| 자료구조 | 추가 | 삭제 | 조회/탐색 | 순서 보장 |
|---|---|---|---|---|
| `ArrayList` | 끝 O(1) | 끝 O(1) / 중간 O(N) | 인덱스 O(1) / 값 O(N) | 삽입 순서 |
| `HashMap`/`HashSet` | O(1) | O(1) | O(1) | 없음 |
| `TreeMap`/`TreeSet` | O(log N) | O(log N) | O(log N) | 정렬됨 |
| `ArrayDeque` (스택/큐) | O(1) | O(1) | 양 끝만 O(1) | 삽입 순서 |
| `PriorityQueue` | O(log N) | O(log N) | top만 O(1) | 우선순위만 |

---

## 정리

- **"빠른 존재 확인·빈도"** → `HashSet`/`HashMap` (O(1)의 마법)
- **"짝/최근/되돌리기"** → 스택(`ArrayDeque`)
- **"순서대로/최단거리"** → 큐(`ArrayDeque`) → BFS
- **"항상 최솟값/최댓값 먼저"** → `PriorityQueue`
- List의 `contains`/값 삭제는 O(N)임을 잊지 말 것
