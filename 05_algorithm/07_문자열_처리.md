# 07. 문자열 처리 — 국내 코딩테스트의 절반은 문자열이다

> **이 문서에서 배우는 것**
> - 자바 문자열의 **기본기와 함정**: char↔int 변환, 주요 메서드, `==` vs `.equals()`
> - **문자 빈도·아나그램**: `int[26]` 패턴 (해시맵보다 빠르고 간단)
> - **회문(팰린드롬)** 검사와 **슬라이딩 윈도우** — 투 포인터의 대표 활용
> - **문자열 파싱**: `split` / `StringTokenizer` / 정규식과, 초심자를 잡는 함정들
> - **문자열 매칭**: `indexOf`부터 KMP 개념까지

문자열은 "어렵다기보다 **함정이 많은**" 영역입니다. `split(".")`이 왜 빈 배열을 주는지, 왜 `==`로 비교하면 틀리는지 —
이걸 모르면 논리는 맞아도 계속 틀립니다. 특히 **카카오·라인 등 국내 기업 코딩테스트는 문자열 파싱 문제를 사랑합니다.**
(신규 아이디 추천, 문자열 압축, 다트 게임… 전부 문자열입니다.)

> 💡 이 문서는 [01(시간복잡도)](./01_기초_시간복잡도와_입출력.md)과 [02(자료구조)](./02_자료구조_활용.md)를 먼저 보면 좋습니다.
> `StringBuilder`, `HashMap`, 투 포인터가 여기서 실제로 쓰입니다.

## 1. 자바 문자열 기본기 — 이것부터 확실히

### ① char와 int는 넘나든다 (아스키 코드)

문자열 문제의 핵심 무기입니다. 문자에서 `'0'`이나 `'a'`를 빼면 **숫자·인덱스**가 됩니다.

```java
char digit = '7';
int n = digit - '0';        // 문자 '7' → 숫자 7

char c = 'c';
int idx = c - 'a';          // 'a'=0, 'b'=1, 'c'=2 ... → 알파벳 인덱스 (0~25)

char back = (char) ('a' + 2); // 인덱스 → 다시 문자 'c'
```

> 📌 이 `c - 'a'`로 얻은 0~25 인덱스가 **`int[26]` 빈도 배열의 핵심**입니다. (2절에서 바로 씁니다)

### ② 자주 쓰는 String 메서드

| 메서드 | 하는 일 | 주의 |
|---|---|---|
| `length()` | 길이 | 배열은 `.length`(괄호X), 문자열은 `.length()` |
| `charAt(i)` | i번째 문자 | 범위 벗어나면 예외 |
| `substring(a, b)` | a부터 b **직전**까지 | `[a, b)` 반열린 구간 |
| `indexOf("x")` | 처음 나오는 위치 | 없으면 `-1` |
| `contains("x")` | 포함 여부 | boolean |
| `toCharArray()` | `char[]`로 변환 | 순회·정렬에 편함 |
| `split(정규식)` | 쪼개기 | **인자가 정규식**! (5절 함정) |
| `replace(a, b)` | 치환 | `replaceAll`은 정규식 |

### ③ 문자열 비교는 반드시 `.equals()`

```java
String a = new String("apple");
String b = new String("apple");

a == b          // false! ← 내용이 아니라 "주소(객체)"를 비교
a.equals(b)     // true   ← 내용 비교 (항상 이걸 쓰세요)
```

> ⚠️ 이건 자바 초심자가 **가장 많이** 틀리는 부분입니다. 문자열 값 비교는 예외 없이 `.equals()`.
> (리터럴 `"apple" == "apple"`은 우연히 true가 나오기도 해서 더 헷갈립니다. 습관을 `.equals()`로 고정하세요.)

### ④ 반복 조작은 StringBuilder (01문서 복습)

문자열은 **불변(immutable)** 이라, 반복문에서 `+=`로 이어붙이면 매번 통째 복사 → **O(N²)**.
반복해서 만들거나 뒤집을 땐 `StringBuilder`:

```java
String reversed = new StringBuilder("algorithm").reverse().toString();  // 뒤집기
```

## 2. 문자 빈도와 아나그램 — `int[26]` 패턴

"어떤 문자가 몇 번 나왔나"는 문자열 문제의 단골입니다. 소문자만이면 **크기 26 배열**이 해시맵보다 빠르고 간단합니다.

```java
int[] cnt = new int[26];
for (char c : s.toCharArray()) cnt[c - 'a']++;   // 'a'~'z' → 0~25
```

> 📌 알파벳 대소문자 모두면 `int[52]`나 `int[128]`(아스키 전체), 한글·유니코드가 섞이면 `HashMap<Character,Integer>`.
> **범위를 먼저 확인**하고 배열 크기를 정하세요.

### 아나그램 판별

두 문자열이 **같은 문자를 같은 개수**로 가지면(순서만 다름) 아나그램입니다. (예: `listen` ↔ `silent`)

```java
static boolean isAnagram(String a, String b) {
    if (a.length() != b.length()) return false;   // 길이 다르면 끝
    int[] cnt = new int[26];
    for (int i = 0; i < a.length(); i++) {
        cnt[a.charAt(i) - 'a']++;   // a는 +1
        cnt[b.charAt(i) - 'a']--;   // b는 -1
    }
    for (int c : cnt) if (c != 0) return false;   // 전부 0이어야 아나그램
    return true;
}
```

전체 예제: [샘플/src/Ex19_CharFrequency_Anagram.java](./샘플/src/Ex19_CharFrequency_Anagram.java)

## 3. 회문(팰린드롬) — 투 포인터

**거꾸로 읽어도 같은** 문자열(예: `level`, `기러기`). 양 끝에서 가운데로 좁혀오며 한 쌍씩 비교합니다. O(N), 추가 메모리 0.

```java
static boolean isPalindrome(String s) {
    int lo = 0, hi = s.length() - 1;
    while (lo < hi) {
        if (s.charAt(lo) != s.charAt(hi)) return false;
        lo++;
        hi--;
    }
    return true;
}
```

실전에서는 **"영문·숫자만 보고 대소문자·공백·기호는 무시"** 하는 버전이 자주 나옵니다.
포인터를 옮기며 `Character.isLetterOrDigit`로 걸러내고 `toLowerCase`로 비교하면 됩니다.
전체 예제: [샘플/src/Ex20_Palindrome.java](./샘플/src/Ex20_Palindrome.java)

> 💡 **투 포인터**는 "양 끝에서 좁히기"(회문)와 "정렬 후 두 값 찾기" 등에 두루 쓰이는 핵심 기법입니다.
> 다음 절의 슬라이딩 윈도우도 포인터 두 개를 쓰는 사촌 격입니다.

## 4. 슬라이딩 윈도우 — 연속 구간 문제의 정석

**"연속된 부분 문자열/구간"** 을 다룰 때, 구간(창)을 옮겨가며 O(N)에 푸는 기법입니다.
왼쪽·오른쪽 포인터로 구간을 유지하고, 조건이 깨지면 왼쪽을 당겨 구간을 줄입니다.

### 대표 예: 중복 없는 가장 긴 부분 문자열

> `"abcabcbb"` → 중복 없이 가장 긴 부분 문자열은 `"abc"`(길이 3).

```java
static int longestUnique(String s) {
    Map<Character, Integer> lastSeen = new HashMap<>();  // 문자 → 마지막 위치
    int left = 0, best = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (lastSeen.containsKey(c) && lastSeen.get(c) >= left) {
            left = lastSeen.get(c) + 1;   // 중복 발생 → 왼쪽을 그 다음으로 점프
        }
        lastSeen.put(c, right);
        best = Math.max(best, right - left + 1);   // 현재 창 길이 갱신
    }
    return best;
}
```

이중 반복(모든 부분 문자열 확인, O(N²))을 **한 번 훑기(O(N))** 로 바꾸는 것이 슬라이딩 윈도우의 힘입니다.
전체 예제: [샘플/src/Ex22_SlidingWindow.java](./샘플/src/Ex22_SlidingWindow.java)

## 5. 문자열 파싱 — split / StringTokenizer / 정규식 (함정 주의!)

국내 코딩테스트 문자열 문제의 **핵심이자 실수 다발 지대**입니다.

### `split` — 인자는 "정규식"이다

```java
"kim,lee,park".split(",")     // [kim, lee, park]  OK
```

**🧨 함정 A — 특수문자:** `split`의 인자는 문자열이 아니라 **정규식**입니다. `.`, `|`, `*` 등은 특별한 뜻을 가집니다.

```java
"192.168.0.1".split(".")      // []  ← '.'은 정규식에서 '아무 문자'라 전부 사라짐!
"192.168.0.1".split("\\.")    // [192, 168, 0, 1]  ← 이스케이프(\\.)해야 진짜 점
```

**🧨 함정 B — 뒤쪽 빈 값 소실:** 기본 `split`은 **뒤쪽의 빈 문자열을 버립니다.** 개수가 중요하면 `limit = -1`.

```java
"a,b,,".split(",")       // [a, b]        길이 2 (뒤 빈 값 사라짐)
"a,b,,".split(",", -1)   // [a, b, , ]    길이 4 (빈 값 유지)
```

> ⚠️ 참고: 문자열이 구분자로 **시작**하면 맨 앞의 빈 문자열은 (뒤쪽과 달리) **유지**됩니다.
> `"!abc".split("!")` → `["", "abc"]`. 앞뒤 규칙이 다르다는 걸 기억하세요.

### `StringTokenizer` — 빠르고 빈 토큰이 안 생김

공백 기준 파싱엔 이게 더 안전하고 빠릅니다. 연속 공백이 있어도 빈 토큰을 만들지 않습니다.

```java
StringTokenizer st = new StringTokenizer("  10   20 30 ");  // 기본 구분자: 공백/탭/개행
while (st.hasMoreTokens()) {
    int x = Integer.parseInt(st.nextToken());   // 10, 20, 30 (빈 토큰 없음)
}
```

### 정규식으로 원하는 문자만 남기기

```java
"abc123!!".replaceAll("[^0-9]", "")     // "123"   (숫자 아닌 것 제거)
"abc123!!".replaceAll("[^a-zA-Z]", "")  // "abc"   (영문자 아닌 것 제거)
```

- `[^...]`는 "이 문자들이 **아닌** 것"이라는 뜻. `[^0-9]` = 숫자가 아닌 모든 문자.

전체 예제: [샘플/src/Ex21_StringParsing.java](./샘플/src/Ex21_StringParsing.java)

## 6. 문자열 매칭 — "찾기"

"큰 글에서 특정 패턴이 어디 있나"는 대부분 **내장 `indexOf`로 충분**합니다.

```java
"abcdefabc".indexOf("abc")     // 0  (처음 위치)
"abcdefabc".indexOf("abc", 1)  // 6  (인덱스 1부터 다시 찾기)
```

직접 브루트포스로 짜면 O(N×M) (본문 길이 N, 패턴 M):

```java
// 개념: 시작 위치를 옮겨가며 한 글자씩 대조
for (int i = 0; i + m <= n; i++) {
    int j = 0;
    while (j < m && text.charAt(i + j) == pattern.charAt(j)) j++;
    if (j == m) { /* i에서 발견 */ }
}
```

> 📌 **KMP(Knuth-Morris-Pratt)** 는 이 매칭을 O(N+M)으로 하는 고급 알고리즘입니다.
> 하지만 **대부분의 코딩테스트는 `indexOf`나 브루트포스로 통과**합니다. KMP는 "문자열 매칭인데 N이 매우 크고
> 시간초과가 날 때"만 꺼내세요. 지금은 "이런 게 있다" 정도로 충분합니다. (심화·선택)

## 7. 문자열 함정 총정리 (틀렸을 때 여기부터 확인)

1. **`==` 로 문자열 비교** → 항상 `.equals()`
2. **`split(".")`, `split("|")`** → 정규식이므로 `\\.`, `\\|` 로 이스케이프
3. **뒤쪽 빈 값이 필요한데 사라짐** → `split(",", -1)`
4. **반복문에서 `String +=`** → `StringBuilder` (O(N²) → O(N))
5. **`length()` vs `length`** → 문자열은 `s.length()`, 배열은 `arr.length`
6. **빈도 배열 크기** → 소문자만 26, 대소문자 52, 아스키 128, 한글·유니코드는 `HashMap`
7. **`substring(a, b)`** 은 b **직전**까지 (b는 미포함)

## 정리

- **char↔int**(`c - '0'`, `c - 'a'`)와 **`int[26]` 빈도 배열**이 문자열 문제의 기본 무기.
- 회문·부분문자열은 **투 포인터 / 슬라이딩 윈도우**로 O(N).
- 파싱은 **split(정규식 함정!)** 과 **StringTokenizer**, 필요한 문자만 **정규식 replaceAll**.
- 매칭은 대개 **`indexOf`로 충분**, KMP는 최후의 수단.
- 비교는 **`.equals()`**, 반복 생성은 **`StringBuilder`** — 예외 없음.
