# 알고리즘 기초 — 코딩테스트를 위한 자바 알고리즘

웹 개발(HTML/CSS/JS → DB → 스프링)을 익힌 여러분이, 이제 **문제를 논리로 푸는 힘**을 기릅니다.
이 모듈은 취업·이직의 관문인 **코딩테스트(코테)** 를 자바로 통과하기 위한 최소한의 알고리즘 지도입니다.

> **이 모듈의 목표**
> - 코테에 자주 나오는 알고리즘 **종류**를 알고, 문제를 보면 "이건 무슨 유형이다"를 판단할 수 있다.
> - 각 유형의 **자바 코드 패턴**을 손에 익혀, 백지에서 재현할 수 있다.
> - 시간초과·메모리초과·틀렸습니다의 **원인을 스스로 진단**할 수 있다.

암기가 아니라 **직접 쳐 보고, 시간초과를 만나 보고, 고쳐 보는 것**이 핵심입니다.
모든 샘플은 [샘플/src](./샘플/src/)에 있고, JDK 17 하나로 바로 실행됩니다.

---

## 왜 알고리즘인가 — 실무와 코테의 관계

| 구분 | 실무 개발 | 코딩테스트 |
|---|---|---|
| 평가 기준 | 유지보수성, 협업, 요구사항 충족 | **정확성 + 제한 시간 내 통과** |
| 흔한 함정 | 과설계 | **시간복잡도 계산 실패 → 시간초과** |
| 얻는 것 | 동작하는 서비스 | 논리적 문제 분해력, 자료구조 감각 |

코테는 실무의 전부가 아니지만, **자료구조를 언제 왜 쓰는지 아는 감각**은 실무에서도 그대로 쓰입니다.
(예: `HashMap`으로 O(1) 조회, `PriorityQueue`로 우선순위 처리 — 스프링 코드에도 매일 등장합니다.)

## 시간복잡도 한눈에 — "제한 시간 안에 되나?"

대부분의 코테는 **1초 ≈ 1억(10^8) 번 연산**을 기준으로 잡습니다. 입력 크기 N을 보고 허용 복잡도를 역산하세요.

| 입력 크기 N | 허용되는 대략적 복잡도 | 대표 접근 |
|---|---|---|
| N ≤ 11 | O(N!) | 순열 완전탐색 |
| N ≤ 25 | O(2^N) | 부분집합 완전탐색 |
| N ≤ 100 | O(N^3) | 3중 반복, 플로이드 |
| N ≤ 1,000 | O(N^2) | 2중 반복, 기본 DP |
| N ≤ 100,000 | O(N log N) | 정렬, 이진탐색, 힙 |
| N ≤ 10,000,000 | O(N) | 한 번 훑기 |

자세한 계산법은 [01번 문서](./01_기초_시간복잡도와_입출력.md)에서 실제 실행 시간과 함께 체감합니다.

3. 채점 사이트 계정 — 아래 "연습 사이트" 참고

## 샘플 코드 실행

[샘플/src](./샘플/src/)에 문서별 예제가 들어 있습니다. 이 폴더에서(PowerShell 기준):

```powershell
javac -encoding UTF-8 -d out src/Ex01_BigO.java
java -cp out Ex01_BigO
```

- macOS/리눅스는 클래스패스 구분자 `;`가 아니라 `:`입니다 (`-cp` 는 단일 클래스라 여기선 차이 없음).
- 각 파일 맨 위 주석에 실행 방법과 무엇을 보는 예제인지 적혀 있습니다.

## 이 모듈이 끝나면

- 문제를 읽고 입력 크기로 **허용 복잡도를 역산**해 접근법을 좁힐 수 있습니다.
- 대표 유형(6개 핵심 + 문자열)의 자바 코드 패턴을 **백지에서 재현**할 수 있습니다.
- "시간초과 / 메모리초과 / 틀렸습니다"를 만났을 때 **어디를 의심할지** 압니다.
