# 06. 동적계획법(DP) — 한 번 푼 문제는 다시 풀지 않는다

> **이 문서에서 배우는 것**
> - **DP의 두 조건**: 최적 부분 구조 + 겹치는 부분 문제
> - **메모이제이션**(위→아래, 재귀+캐시) vs **타뷸레이션**(아래→위, 반복문+표)
> - **점화식(DP 식)** 을 세우는 사고 과정 — DP의 90%는 여기
> - 대표 문제 3종: 피보나치, 0/1 배낭, LIS

DP는 코테에서 가장 **"발상이 어렵다"** 고 느끼는 유형입니다. 하지만 정체는 단순합니다:
**"완전탐색인데 중복 계산이 너무 많아서, 결과를 저장해 재사용하는 것."**
많은 문제를 풀며 점화식 세우는 감각을 기르는 게 유일한 길입니다.

> 💡 [04번 완전탐색](./04_완전탐색과_그리디.md)을 먼저 이해했다고 가정합니다.
> DP는 "완전탐색의 시간초과를 캐싱으로 뚫는 것"이라, 완전탐색을 알아야 DP가 보입니다.

---

## 1. DP가 필요한 순간 — 피보나치로 보는 중복

피보나치를 순진하게 재귀로 짜면:

```java
static int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);   // 같은 값을 몇 번이고 다시 계산!
}
```

`fib(5)`를 부르면 `fib(3)`이 2번, `fib(2)`가 3번... **같은 계산이 지수적으로 반복**됩니다. → O(2^N), n=50이면 사실상 멈춤.

**해법: 한 번 구한 답을 저장(메모)해 두고 재사용.** 이게 DP의 전부입니다.

DP가 통하는 문제의 두 조건:

| 조건 | 의미 |
|---|---|
| **겹치는 부분 문제** | 같은 작은 문제를 반복해서 풀게 됨 (위 fib처럼) |
| **최적 부분 구조** | 작은 문제의 (최적) 답으로 큰 문제의 답을 만들 수 있음 |

---

## 2. 두 가지 구현 방식

### 방식 ① 메모이제이션 (Top-Down, 위→아래)

재귀는 그대로 두되, **결과를 배열에 저장**하고 있으면 즉시 반환.

```java
static long[] memo;

static long fib(int n) {
    if (n <= 1) return n;
    if (memo[n] != 0) return memo[n];         // 이미 구했으면 재사용
    return memo[n] = fib(n - 1) + fib(n - 2); // 구하면서 저장
}
// memo = new long[n+1]; 로 초기화 후 호출
```

- 장점: 완전탐색 재귀에서 **캐시 한 줄만 추가**하면 되어 직관적.
- 단점: 재귀가 깊으면 StackOverflow 위험.

### 방식 ② 타뷸레이션 (Bottom-Up, 아래→위)

작은 값부터 **표(배열)를 채워 올라갑니다.** 재귀 없이 반복문으로.

```java
static long fib(int n) {
    long[] dp = new long[n + 1];
    dp[0] = 0; dp[1] = 1;
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + dp[i - 2];   // 앞의 답으로 현재 답을 만듦
    }
    return dp[n];
}
```

- 장점: StackOverflow 없음, 대체로 더 빠름.
- 단점: 채우는 **순서**를 스스로 정해야 함.

> 📌 둘 중 뭘 쓰든 결과는 같습니다. 처음엔 완전탐색에서 자연스럽게 넘어가는 **메모이제이션**이 배우기 쉽고,
> 익숙해지면 **타뷸레이션**이 안전하고 빠릅니다. 세 예제로 실제 시간 차이를 [Ex15](./샘플/src/Ex15_Fibonacci.java)에서 봅니다.

## 3. DP의 핵심 — 점화식 세우기

DP 문제의 90%는 **"dp[i]를 무엇으로 정의하고, 어떻게 앞의 값으로 표현할까?"** 를 찾는 것입니다.
이 사고 과정을 두 대표 문제로 익힙니다.

### 대표 예 1: 계단/타일 채우기류 (1차원 DP)

> "2×n 바닥을 1×2 타일로 채우는 경우의 수" → 사실 피보나치와 같은 점화식.
> `dp[i] = dp[i-1] + dp[i-2]` (마지막 타일을 세로로 놓느냐 가로 2개로 놓느냐).

**점화식 세우는 질문**: "마지막 한 칸(또는 한 선택)을 기준으로 경우를 나누면?" 이 질문이 대부분의 1차원 DP를 엽니다.

### 대표 예 2: 0/1 배낭 문제 (2차원 DP)

> 무게 한도 W인 가방에, (무게·가치)를 가진 물건들을 넣어 **가치 합 최대**를 만들기. 각 물건은 0개 또는 1개.

- `dp[i][w]` = "앞에서 i개 물건까지 고려했고, 무게 한도가 w일 때의 최대 가치"
- 각 물건에 대해 **넣거나 / 안 넣거나** 두 선택:

```java
// 안 넣음: dp[i-1][w] 그대로
// 넣음:    dp[i-1][w - weight[i]] + value[i]   (넣을 수 있을 때만)
dp[i][w] = Math.max(
    dp[i-1][w],
    (w >= weight[i]) ? dp[i-1][w - weight[i]] + value[i] : 0
);
```

> 💡 이 "i번째를 넣냐 마냐"로 상태를 쪼개는 발상이 배낭 DP의 핵심입니다.
> [04번](./04_완전탐색과_그리디.md)에서 본 "그리디로 안 되는 거스름돈/배낭"이 바로 이 DP로 풀립니다.

전체 예제: [샘플/src/Ex16_Knapsack.java](./샘플/src/Ex16_Knapsack.java)

### 대표 예 3: LIS (최장 증가 부분 수열)

> 수열에서 "앞에서부터 골랐을 때 계속 커지는" 가장 긴 부분 수열의 길이.
> 예: [10, 20, 10, 30, 20, 50] → [10, 20, 30, 50] 길이 4.

- `dp[i]` = "i번째 원소로 **끝나는** 증가 수열의 최대 길이"
- `dp[i] = max(dp[j] + 1)` (단, `j < i` 이고 `arr[j] < arr[i]` 인 모든 j 중)

```java
int[] dp = new int[n];
Arrays.fill(dp, 1);              // 자기 자신만으로도 길이 1
for (int i = 0; i < n; i++)
    for (int j = 0; j < i; j++)
        if (arr[j] < arr[i])
            dp[i] = Math.max(dp[i], dp[j] + 1);
// 답 = dp 배열의 최댓값
```

이 O(N^2) 풀이가 기본이고, N이 크면 이진탐색으로 O(N log N)까지 최적화합니다.
전체 예제: [샘플/src/Ex17_LIS.java](./샘플/src/Ex17_LIS.java)

## 4. DP 문제 접근 순서 (막힐 때 이 순서로)

1. **완전탐색으로 먼저 생각해 본다** — 모든 경우를 재귀로 어떻게 셀까?
2. 거기서 **같은 인자로 반복 호출되는 부분**이 보이는가? → DP 가능 신호
3. **상태(dp의 인덱스)를 정의**한다 — "dp[i] 또는 dp[i][j]가 뜻하는 바"를 한 문장으로.
4. **점화식**을 세운다 — 마지막 선택/칸을 기준으로 경우를 나눠 앞의 값으로 표현.
5. **초기값(base case)** 과 **채우는 순서**를 정한다.
6. 답이 dp의 어디에 있는지 확인한다 (dp[n]? dp 전체의 max?).

> ⚠️ **오버플로 주의**: DP는 경우의 수·합을 누적하다 int(21억)를 넘기 쉽습니다.
> "값이 커질 수 있다" 싶으면 `long`, 문제에서 "나머지로 출력"하라면 매 단계 `% 1_000_000_007`.

---

## 정리

- DP = **완전탐색 + 결과 저장(캐시)**. 겹치는 부분 문제 + 최적 부분 구조가 조건.
- **메모이제이션**(재귀+캐시)과 **타뷸레이션**(반복+표)은 같은 것의 두 방식.
- 핵심은 **상태 정의 + 점화식**. "마지막 선택 기준으로 나누기"가 실마리.
- 누적값 오버플로(→ `long` / 나머지 연산)를 항상 경계.
