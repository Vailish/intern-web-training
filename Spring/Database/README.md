# 스프링부트 4 + JPA — H2에서 MySQL까지, CRUD 완성하기

[스프링부트 교육](../SpringBoot/README.md)에서 자기소개서를 **등록하고 조회**하는 앱을 만들어 봤습니다.
이번 모듈에서는 최신 **스프링부트 4**로 프로젝트를 새로 만들어,

1. **수정과 삭제까지 갖춘 완전한 CRUD**를 완성하고
2. 개발용 DB(**H2**)에서 진짜 서버 DB(**MySQL**)로 **코드 한 줄 안 바꾸고** 갈아타 보고
3. 같은 앱을 회사 실무 스택인 **MyBatis**로 다시 만들며 JPA와 비교

까지 해 봅니다. "개발은 가벼운 DB로, 운영은 서버 DB로"는 실무에서 매일 쓰는 패턴이고,
MyBatis는 여러분이 입사 후 실제로 만질 기술입니다.

## 시작 전 준비물

1. **[스프링부트 교육](../SpringBoot/README.md) 완주** — 컨트롤러/서비스/리포지토리 구조와 타임리프를 안다고 가정합니다.
2. **[데이터베이스 교육](../../03_Database/README.md) 완주** — 특히 02(접속과 권한), 04(JDBC)를 알아야 "JPA가 대신해 주는 일"이 보입니다.
3. **JDK 17 이상** — `java -version`으로 확인.
4. (02 문서만) **Docker Desktop** — [DB 교육 05](../../03_Database/05_심화_도커로_MySQL.md)에서 쓴 MySQL 컨테이너를 다시 사용합니다.

> 📌 **스프링부트 교육은 3.5, 여기는 4?**
> 스프링부트 교육 자료를 만들 때는 안정성을 위해 3.5 계열을 썼습니다.
> 지금은 [start.spring.io](https://start.spring.io)의 기본 선택이 4.x라서, 이번에는 **기본값 그대로** 진행합니다.
> 배우는 개념은 같고, 달라진 부분은 01 문서에서 짚어 줍니다.

## 완성 샘플 프로젝트

**실행 가능한 완성본** 두 개가 들어 있습니다. (둘 다 빌드·CRUD 동작 검증 완료)

| 폴더 | 내용 |
|---|---|
| [샘플/intro-jpa](./샘플/intro-jpa/) | 01~02 문서의 완성본 (JPA) |
| [샘플/intro-mybatis](./샘플/intro-mybatis/) | 03 문서의 완성본 — 같은 앱, 리포지토리 계층만 MyBatis |

```powershell
cd 샘플/intro-jpa        # 또는 샘플/intro-mybatis

# H2로 실행 (기본)
.\gradlew.bat bootRun

# MySQL로 실행 (도커 MySQL이 켜져 있어야 합니다 → 02 문서)
.\gradlew.bat bootRun --args='--spring.profiles.active=mysql'
```

브라우저에서 `http://localhost:8080` 접속. 종료는 콘솔에서 `Ctrl + C`.
(첫 실행은 그레이들과 라이브러리를 내려받느라 몇 분 걸립니다.)

## 이 모듈이 끝나면

- 등록/조회만 있던 앱에 **수정·삭제를 스스로 붙일 수 있습니다** (실무 CRUD의 완성형).
- `save()` 없이 값만 바꿔도 UPDATE가 나가는 **변경 감지(Dirty Checking)** 를 설명할 수 있습니다.
- **프로필** 하나로 개발 DB와 운영 DB를 갈아끼우는 방법을 알게 됩니다 — eGovFrame 실무에서 개발/운영 설정을 나누는 것과 같은 원리입니다.
- JPA와 MyBatis의 차이를 설명할 수 있고, **회사 실무 코드(MyBatis 매퍼 XML)를 읽을 수 있게** 됩니다.
