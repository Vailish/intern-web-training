# 01. 외부 API 호출이란 — 서버가 "손님"이 되는 순간

> **이 문서에서 배우는 것**
> - "우리 서버가 남의 서버를 부른다"는 것이 무슨 뜻인지
> - 왜 실무는 남이 만든 API를 갖다 쓰는지 (직접 안 만들고)
> - 스프링부트의 HTTP 클라이언트 3형제: **RestClient / RestTemplate / WebClient** 비교와 선택
> - 우리가 쓸 외부 API(Open-Meteo)의 생김새

---

## 1. 지금까지와 무엇이 다른가

지금까지 브라우저와 우리 서버의 관계는 이랬습니다.

```
[브라우저] ──── 요청 ────▶ [우리 스프링부트]   (우리는 "받는 쪽" = 서버)
          ◀─── 응답 ─────
```

이번에는 우리 서버가 **또 다른 서버에게 요청을 보내는** 관계가 추가됩니다.

```
[브라우저] ──▶ [우리 스프링부트] ──── 요청 ────▶ [외부 API 서버]   (우리는 "보내는 쪽" = 클라이언트)
                              ◀─── 응답(JSON) ──
```

**같은 프로그램이 상황에 따라 서버도 되고 클라이언트도 됩니다.**
브라우저를 상대할 땐 서버, 외부 API를 상대할 땐 클라이언트. 이 두 역할은 동시에 성립합니다.

> 📌 **"API를 만든다" vs "API를 쓴다"** — Front-Back 모듈에서는 우리가 REST API를 **만들었습니다**
> (`@RestController`). 이번엔 남이 만든 REST API를 우리가 **씁니다**(`RestClient`).
> 만드는 법을 배웠으니, 쓰는 법도 배우면 양쪽을 다 이해하게 됩니다.

---

## 2. 왜 남의 API를 갖다 쓰나 — "만들지 않는 것도 실력"

날씨 데이터를 직접 만들려면 전국에 관측소를 세우고 위성을 띄워야 합니다. 불가능하죠.
그래서 **이미 잘 만들어 둔 곳의 데이터를 API로 받아다 씁니다.** 실무의 대부분이 이렇습니다.

| 하고 싶은 일 | 직접 만들면 | 실무에서는 (외부 API) |
|---|---|---|
| 결제 처리 | 카드사·은행과 직접 계약·연동 (수개월) | 토스페이먼츠/아임포트 등 **PG사 API** |
| 지도 표시 | 전 세계 지도 데이터 구축 (불가능) | 카카오/네이버/구글 **지도 API** |
| 문자·알림톡 발송 | 통신사 회선 계약 | 네이버 클라우드/알리고 **메시징 API** |
| 공공 정보(공휴일·주소·기상) | 정부 데이터 수집 (불가능) | **공공데이터포털(data.go.kr) API** |
| 소셜 로그인 | 자체 회원 시스템 | 카카오/구글 **OAuth API** |

> 🏢 **eGovFrame 연결 포인트** — 공공기관 SI 프로젝트에서는 **공공데이터포털(data.go.kr)** 과
> 행정안전부의 각종 연계 API를 정말 자주 붙입니다. eGovFrame에도 외부 연계용 유틸이 있지만,
> 원리는 이 문서와 똑같습니다 — "HTTP로 요청 보내고, 받은 XML/JSON을 객체로 바꾼다."
> 그 실무 API(키 기반) 연동은 [03 문서](./03_에러처리와_실무_고려사항.md)에서 다룹니다.

---

## 3. 외부 API는 어떻게 부르나 — 결국 HTTP 요청 한 번

거창해 보이지만, 외부 API 호출은 **브라우저 주소창에 URL을 치는 것과 똑같습니다.**
아래 주소를 브라우저에 그대로 넣어 보세요. JSON이 그대로 보입니다.

```
https://api.open-meteo.com/v1/forecast?latitude=37.57&longitude=126.98&current=temperature_2m
```

```json
{
  "latitude": 37.55,
  "longitude": 127.0,
  "current": { "time": "2026-07-21T13:00", "temperature_2m": 25.5 }
}
```

우리가 할 일은 이 요청을 **자바 코드로** 보내고, 돌아온 JSON을 **자바 객체로** 바꾸는 것뿐입니다.
그 도구가 **RestClient** 입니다.

---

## 4. HTTP 클라이언트 3형제 — 무엇을 쓸 것인가

스프링에는 외부 API를 부르는 도구가 시대순으로 셋 있습니다. **결론부터: 지금은 `RestClient`.**

| 도구 | 등장 | 방식 | 지금 선택? |
|---|---|---|---|
| **RestTemplate** | 오래됨(Spring 3) | 동기(기다림) | ❌ 유지보수 모드. 신규 코드엔 권장 안 함 (하지만 실무 레거시에 아주 많음) |
| **WebClient** | Spring 5 | 비동기/반응형 | △ 대량 동시 호출·스트리밍이 필요할 때. 개념이 어려움(Mono/Flux) |
| **RestClient** | **Spring 6.1 / 부트 3.2** | 동기(기다림) | ✅ **오늘의 표준.** RestTemplate처럼 쉬우면서 문법이 현대적 |

- **RestClient** 는 "RestTemplate의 쉬움 + WebClient의 깔끔한 문법"을 합친 최신 도구입니다.
  대부분의 웹 앱은 "요청 보내고 응답 올 때까지 기다렸다가 화면 그림" = **동기**면 충분하므로 RestClient가 정답입니다.
- **WebClient**(비동기)는 "수천 개를 동시에 부르며 기다리는 동안 다른 일 하기"가 필요할 때만 씁니다.
  우리 예제(한 요청 처리에 API 두 번)에는 과합니다.

```java
// 같은 일을 세 도구로 (문법 비교)

// RestTemplate (레거시)
Weather w = restTemplate.getForObject(url, Weather.class);

// WebClient (비동기 — .block()으로 억지로 기다리면 WebClient 쓰는 의미가 없음)
Weather w = webClient.get().uri(url).retrieve().bodyToMono(Weather.class).block();

// RestClient (오늘의 표준) ← 우리가 쓸 것
Weather w = restClient.get().uri(url).retrieve().body(Weather.class);
```

> ⚠️ **함정 — "그럼 RestTemplate은 몰라도 되나요?"** 아닙니다. 실무 **레거시 코드의 90%가 RestTemplate**
> 입니다. 회사 코드에서 `restTemplate.getForObject(...)`, `exchange(...)` 를 보면 "아, 외부 API 부르는
> 옛날 방식이구나" 하고 알아볼 수 있어야 합니다. **새로 짤 땐 RestClient, 읽을 땐 RestTemplate도.**

> 📌 **왜 3개나 있나?** — 자바 웹의 역사입니다. RestTemplate은 너무 오래돼 문법이 낡았고,
> 그걸 고치려 만든 WebClient는 비동기라 배우기 어려웠습니다. "동기인데 문법은 현대적인 것"의
> 수요가 커서 RestClient가 나왔습니다. 신입은 **RestClient 하나만 제대로** 익히면 됩니다.

## 5. 우리가 쓸 API — Open-Meteo (무료·API키 불필요)

- **주소 검색(지오코딩):** `https://geocoding-api.open-meteo.com/v1/search`
  → 도시 이름을 위도·경도로 바꿔 줍니다.
- **날씨 예보:** `https://api.open-meteo.com/v1/forecast`
  → 위도·경도의 현재 날씨를 줍니다.

**API 키가 필요 없어서** 회원가입·발급 절차 없이 바로 실습할 수 있습니다. (교육용으로 최적)
실무 API 대부분은 키가 필요한데, 그 키를 안전하게 다루는 법은 03 문서에서 배웁니다.

> 📌 **출처 신뢰도** — Open-Meteo는 유럽 국가 기상청들의 공개 데이터를 모아 제공하는 서비스입니다.
> 교육·비상업 용도 무료. 다만 "무료 공개 API"는 **예고 없이 스펙이 바뀌거나 느려질 수 있다**는
> 점을 늘 염두에 두세요(그래서 03 문서에서 타임아웃·예외 처리를 배웁니다). 상용 서비스라면
> SLA(가용성 보장)가 있는 유료 API나 공식 기관 API를 검토해야 합니다.

## 정리

- 우리 서버가 **남의 서버를 부르는 클라이언트**가 되는 것 — 그게 외부 API 호출입니다.
- 실무는 대부분 남이 만든 API를 갖다 씁니다. **안 만드는 판단**도 실력입니다.
- 도구는 **RestClient**(오늘의 표준) 하나면 됩니다. RestTemplate은 레거시에서 읽을 줄만 알면 OK.
